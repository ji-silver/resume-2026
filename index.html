Zod 및 React-Hook-Form 연동을 통한 폼 검증 로직 최적화

  • 기존 UI 컴포넌트에 수동으로 주입하던 정규식 및 커스텀 검증 로직을 Zod 스키마 기반으로 중앙화
  • 다수의 입력 필드가 존재하는 복잡한 정산 폼 등에서 필수 검증 누락에 따른 휴먼 에러를 사전 차단하여 코드 안정성 및 유지보수성 향상
문제 인식
  • 수십 개의 입력 필드가 있는 정산 폼을 개발할 때, 각 UI 컴포넌트의 register 속성에 정규식이나 커스텀 검증 함수를 개발자가 일일이 수동으로 주입해야 했음
  • 폼이 복잡해질수록 개발자의 실수로 특정 필드에 필수 검증 로직(예: '숫자만 입력 허용' 등)을 누락하는 휴먼 에러가 빈번하게 발생
기존 코드 (Before)
<DetailInput
  register={form.register('businessField', {
    ...Object.assign(
      formValidate.empty('업태는 필수 정보 입니다'),
      formCount('업태명이 너무 깁니다. (최대 35글자)', 0, 35),
      formValidate.text('문자, 숫자, 공백, 일부 특수문자만 입력 가능합니다.')
    ),
  })}
/>

→ 개발자가 register에 검증 함수를 직접 조합해야 하므로, 필드가 많아질수록 누락 위험이 증가

해결 방법
  • react-hook-form에 Zod 스키마를 결합하여, 뷰(UI) 컴포넌트 내부에서 검증 로직을 완전히 분리
  • 필드별 검증 룰을 최상단의 Zod 스키마(z.object) 한 곳에서만 명세하여, 개발자가 개별 인풋 컴포넌트에 검증 함수를 달아주는 것을 깜빡하더라도 Zod 스키마가 폼 제출을 시스템적으로 차단
성과
  • 폼 검증 관련 휴먼 에러 원천 차단 — 스키마에 정의되지 않은 데이터는 제출 자체가 불가능
  • 새로운 폼 개발 시 검증 로직 작성 시간 단축 및 코드 리뷰 포인트 감소
  • 검증 로직의 단일 소스(Zod 스키마)로 인해 유지보수성과 일관성 대폭 향상

복잡한 단일 폼 UI의 탭 구조 개편 및 렌더링 최적화

  • 방대한 데이터가 나열된 기존의 단일 폼 화면을 사용자의 업무 워크플로우에 맞춰 논리적인 탭 구조로 전면 개편
  • 긴 입력 화면으로 인한 사용자 피로도를 낮춰 UX를 개선하고, 안정적인 폼 상태 유지 및 화면 렌더링 성능 최적화 달성

프론트엔드 에러 메시지 통합 관리 모듈 구현 및 UX 개선

  • 백엔드에서 반환되는 예외 상태 및 에러 코드를 프론트엔드 규격에 맞춰 재가공하는 전용 예외 처리 훅 구현
