클라우드 빌링 솔루션 기능 개선 및 운영

2024.03 ~ 현재 · ㈜씨에이웨이브

기존 클라우드 빌링 솔루션의 운영 안정성과 사용자 편의성 개선 및 백오피스 기능 개발

RBAC 기반 관리자 어드민 페이지 신규 구축

  • 관리자 역할에 따라 접근 가능한 권한과 자원을 분리하여 할당할 수 있는 계층형 권한 관리 시스템의 프론트엔드 화면 구현
  • 라우터 및 UI 컴포넌 레벨에서 렌더링을 동적으로 제어하여 시스템 보안성을 강화하고 관리자 계정 운영의 효율성 증대

OTP 기반 2차 인증(2FA) 등록 및 관리 프로세스 고도화

  • 기존 운영 중인 OTP 기반 2차 인증 시스템에 대한 사용자 인증 수단 등록, 조회 및 해제 관리 화면 구현
  • 인증 수단 관리 프로세스를 UI로 제공하여 사용자 및 관리자의 계정 보안 유지보수 편의성 개선

Zod 및 React-Hook-Form 연동을 통한 폼 검증 로직 최적화

  • 프론트엔드 팀원들과 협업하여 기존 UI 컴포넌트에 수동으로 주입하던 검증 로직을 Zod 스키마 기반으로 중앙화 및 통합 구축
  • 다수의 입력 필드가 존재하는 복잡한 정산 폼 등에서 필수 검증 누락에 따른 휴먼 에러를 사전 차단하여 코드 안정성 및 유지보수성 향상
문제 인식
  • 수십 개의 입력 필드가 있는 복잡한 정산 폼 개발 시, 각 UI 컴포넌트에 개발자가 일일이 커스텀 검증 함수를 수동으로 주입해야 했음
  • 폼이 복잡해질수록 개발자의 실수로 필수 검증 로직(예: 숫자만 입력 등)을 누락하는 휴먼 에러가 빈번하게 발생
기존 코드의 한계 (Before) 
<DetailInput
  register={form.register('businessField', {
    ...Object.assign(
      formValidate.empty('업태는 필수 정보 입니다'),
      formCount('업태명이 너무 깁니다. (최대 35글자)', 0, 35),
      formValidate.text('문자, 숫자, 공백, 일부 특수문자만 입력 가능합니다.')
    ),
  })}
/>

→ 개발자가 register에 검증 함수를 직접 조합해야 하므로, 필드가 많아질수록 누락 위험이 증가

접근 방식
  • react-hook-form에 Zod 스키마를 결합하여, 뷰(UI) 컴포넌트 내부에서 검증 로직을 완전히 분리
  • 필드별 검증 룰을 최상단의 Zod 스키마(z.object) 한 곳에서 명세하여 중앙집중식 검증 체계 설계
결과
  • 스키마에 정의되지 않은 데이터는 제출 자체가 불가능해져 폼 검증 관련 휴먼 에러 원천 차단 (버그 리포트 대폭 감소)
  • 검증 로직 재사용 및 통합을 통해 신규 폼 개발 시 발생하는 반복적인 코드를 제거하고 개발 효율성 향상

복잡한 단일 폼 UI의 탭 구조 개편 및 렌더링 최적화

  • 방대한 데이터가 나열된 기존의 단일 폼 화면을 사용자의 업무 워크플로우에 맞춰 논리적인 탭 구조로 전면 개편
  • 방대한 입력 필드를 논리적인 단위로 분리하고, 탭 전환 시에도 기존 입력 데이터가 안전하게 유지되도록 상태 관리를 구현하여 사용자 피로도 감소 및 입력 편의성 증대

프론트엔드 에러 메시지 통합 관리 모듈 구현 및 UX 개선

  • 백엔드에서 반환되는 예외 상태 및 에러 코드를 프론트엔드 규격에 맞춰 재가공하는 전용 예외 처리 훅 구현
  • 장애 발생 시 불필요한 개발자 용어 노출을 차단하고 직관적인 사용자 안내 제공
